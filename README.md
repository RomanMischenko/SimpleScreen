# <img src="docs/icon.png" width="36" height="36"> SimpleScreen

Скриншот‑инструмент для macOS, размещённый в строке меню. Личный пет‑проект — не предназначен для широкого распространения.

## Назначение

Стандартные сочетания клавиш для создания скриншотов в macOS (`⌘⇧3` / `⌘⇧4`) позволяют либо сохранить PNG‑файл на рабочий стол, либо (при зажатом `Ctrl`) скопировать изображение в буфер обмена. **Одновременно выполнить оба действия невозможно**: приходится либо искать файл в Finder после копирования, либо вручную копировать сохранённый PNG, чтобы вставить его в чат или таск‑трекер.

SimpleScreen решает эту проблему: режим **Save & Copy** (активирован по умолчанию) с помощью одного сочетания клавиш сохраняет скриншот в указанную папку и одновременно копирует его в буфер обмена.

**Основные возможности:**

* настраиваемая папка для сохранения скриншотов;
* переназначаемые сочетания клавиш с проверкой конфликтов с системными;
* автозапуск при входе в систему через `SMAppService`;
* отсутствие иконки в Dock — приложение работает исключительно из строки меню.

## Обзор кода

### Точка входа

* `SimpleScreenApp.swift` — `@main` SwiftUI App, подключает `AppDelegate`, скрывает иконку в Dock через политику активации `.accessory`.
* `AppDelegate.swift` — инициализирует все сервисы, при запуске проверяет наличие разрешения на запись экрана (Screen Recording) и отслеживает его предоставление системой.

### Сервисы

* `Preferences/AppSettings.swift` — `@Observable` единый источник данных для всех пользовательских настроек; читает и записывает значения в `UserDefaults` с префиксом `ss_`.
* `HotKeys/HotKeyManager.swift` — обёртка над Carbon `RegisterEventHotKey`; обрабатывает сочетания `⌘⇧3` (полный экран) и `⌘⇧4` (выбранная область).
* `Notifications/NotificationManager.swift` — обёртка над `UNUserNotificationCenter`; отображает уведомления после каждого захвата экрана.

### Механизм захвата

* `Capture/CaptureEngine.swift` — `@Observable`; использует `SCScreenshotManager` (ScreenCaptureKit) для полноэкранного и регионального захвата. В зависимости от `AppSettings.postCaptureAction` сохраняет изображение в файл, копирует в буфер обмена или выполняет оба действия. При ошибке записи в основную папку автоматически использует `~/Desktop`.
* `Capture/AreaSelectionWindow.swift` — borderless `NSWindow` на уровне `.screenSaver` поверх основного дисплея. `SelectionView` обрабатывает выделение области мышью с отображением размеров в пикселях в реальном времени, отменой по клавише Escape и ограничением минимального размера (10×10 пикселей).

### Пользовательский интерфейс

* `Menu/StatusBarController.swift` — управляет `NSStatusItem` (иконка камеры), формирует и поддерживает выпадающее меню, реактивно обновляет сочетания клавиш через `withObservationTracking`.
* `Preferences/PreferencesView.swift` — SwiftUI `Form` внутри плавающего `NSPanel`; позволяет настроить действие после захвата, папку сохранения, сочетания клавиш (с проверкой конфликтов) и автозапуск.

**Поток данных:** `AppSettings` служит центральным хранилищем настроек, за изменениями которого следят `CaptureEngine`, `StatusBarController` и `PreferencesView`. Весь процесс захвата управляется через `CaptureEngine`. Обработку сочетаний клавиш выполняет `HotKeyManager`, а уведомлений — `NotificationManager`. Прямой доступ к `UserDefaults` или `SCScreenshotManager` из других частей кода отсутствует.

## Требования

* Xcode 16+;
* macOS 15.6+;
* Apple Developer аккаунт (требуется для подписи; для локальной сборки достаточно Personal Team).

## Сборка и запуск

1. Откройте `SimpleScreen.xcodeproj` в Xcode.
2. Выберите схему `SimpleScreen` и Mac в качестве цели запуска (run destination).
3. Нажмите **⌘R**.

После запуска приложение появится в строке меню (иконка камеры). Иконка в Dock отсутствует.

### Разрешения при первом запуске

При первом запуске приложение проверяет наличие разрешения на запись экрана (Screen Recording):

* Если разрешение уже выдано, приложение готово к работе.
* Если разрешения нет, отображается предупреждение с кнопкой **Open System Settings**. Для предоставления доступа перейдите в **System Settings → Privacy & Security → Screen Recording**.
* Приложение проверяет статус разрешения раз в секунду — захват экрана активируется автоматически после его предоставления, перезапуск не требуется.

## Использование

### Захват экрана

| Действие | Пункт меню | Сочетание клавиш по умолчанию |
|--------|-----------|-------------------------------|
| Захват всего экрана | Иконка камеры → Capture Full Screen | **⌘⇧3** |
| Захват выделенной области | Иконка камеры → Capture Selected Area | **⌘⇧4** |

**Выделение области:** поверх основного дисплея появляется crosshair‑оверлей. С помощью мыши растяните прямоугольник — рядом с курсором в реальном времени отображаются размеры в пикселях. После отпускания кнопки мыши выполняется захват. Нажмите **Escape** для отмены. Выделения меньше 10×10 пикселей отклоняются с отображением предупреждающего уведомления.

Если во время выполнения одного захвата запускается другой, он игнорируется без дополнительных сообщений.

### Действия после захвата

Настраиваются в Preferences. Доступны три режима:

| Режим | Действие |
|------|----------|
| Save to File | PNG‑файл сохраняется в указанную папку |
| Copy to Clipboard | Изображение копируется в буфер обмена |
| Save & Copy (по умолчанию) | Выполняются оба действия |

После каждого захвата отображается уведомление с результатом. Файлы именуются по шаблону `Screenshot YYYY-MM-DD at HH.MM.SS.png`. Если основная папка недоступна, приложение использует `~/Desktop` и показывает предупреждение.

### Настройки (Preferences)

Открываются через **Иконка камеры → Preferences…**

* **Post‑Capture Action** — выбор между сохранением, копированием или выполнением обоих действий.
* **Save Location** — кнопка **Choose…** позволяет указать любую папку; по умолчанию используется `~/Pictures/Screenshot/`.
* **Keyboard Shortcuts** — кнопка **Record** рядом с пунктами Full Screen или Area Select. Нажмите нужное сочетание клавиш — конфликты с системными комбинациями автоматически обнаруживаются и отображаются в интерфейсе.
* **Launch at Login** — переключатель для регистрации/удаления приложения в `SMAppService`.

Все настройки сохраняются между запусками.

## Логирование

SimpleScreen использует систему унифицированного логирования macOS (`os.Logger`). Файлы в `/tmp` или домашней директории не создаются — macOS автоматически управляет ротацией и удалением журналов.

### Где посмотреть логи

* **Console.app**: откройте приложение и введите в поиске `subsystem:com.simplescreenapp.SimpleScreen`.
* **Терминал**: выполните команду:  
```
log show --predicate 'subsystem == "com.simplescreenapp.SimpleScreen"' --debug --last 1h
```

### Категории логирования

* `capture` — события захвата: папка сохранения, размеры изображения, успешные операции записи в файл/буфер, ошибки.
* `areaSelect` — события окна выделения: открытие/закрытие, перемещение мыши, отмена по Escape, слишком маленькое выделение.

### Уровни детализации

По умолчанию macOS отображает сообщения уровней `info` и `error`. Для подробной диагностики используйте флаг `--debug` в команде `log show` или выберите **Action → Include Debug Messages** в Console.app.

## Распространение

SimpleScreen разработан для личного использования — готового пайплайна для нотариального подписания релизов нет. Установка выполняется из локально собранного `.app`:

1. Соберите версию Release в Xcode (**Product → Build** в Release‑схеме или с помощью команды):  
```
xcodebuild -project SimpleScreen.xcodeproj -scheme SimpleScreen -configuration Release build
```
2. Переместите `SimpleScreen.app` из `DerivedData/.../Build/Products/Release/` в `/Applications` (или `~/Applications`).
3. При первом запуске Gatekeeper может заблокировать запуск: сборка использует локальную подпись ad‑hoc (`Sign to Run Locally`). Щёлкните правой кнопкой мыши → **Open** → подтвердите **Open** в диалоговом окне (требуется один раз).
4. Предоставьте разрешение **Screen Recording** в **System Settings → Privacy & Security → Screen Recording**.

### Особенности локальной подписи

* Функция **Launch at Login** через `SMAppService` для ad‑hoc сборок может не работать — в этом случае приложение отобразит уведомление об ошибке.
* Локальная подпись привязана к текущей ad‑hoc identity на этой машине. После крупного обновления macOS или сброса identity приложение, возможно, потребуется пересобрать.
* **Не копируйте** `SimpleScreen.app.dSYM` рядом с `.app` — это пакет с отладочными символами, который Xcode создаёт рядом с бинарником (для символикации сбоев), а не часть пользовательского приложения. Его можно безопасно удалить локально — он будет пересоздан при следующей сборке.
